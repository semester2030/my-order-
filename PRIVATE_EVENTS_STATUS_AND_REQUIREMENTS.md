# المناسبات الخاصة — الحالة والمتطلبات

## ملخص تنفيذي

شاشة "المناسبات الخاصة" **موجودة** وتفتح Feed مقدمي الخدمة من فئة `private_events`، لكن **لا يوجد تدفق مخصص** للمناسبات (زواج، تخرج، ملكة، خطوبة) ولا لنوع الخدمات (بوفيه، حلويات، عصائر، مقدمي خدمة).

---

# ١. ما هو منفذ

## Backend

| العنصر | الحالة | الموقع |
|--------|--------|--------|
| حقل `provider_category` في Vendor | ✅ | `vendors.provider_category` — قيم: home_cooking, popular_cooking, **private_events**, grilling |
| تسجيل مقدم خدمة بفئة private_events | ✅ | `RegisterVendorDto.providerCategory` |
| فلتر Feed حسب category | ✅ | `FeedService.getFeed()` — `vendorWhere.providerCategory = category` |
| عرض مقدمي private_events في Feed | ✅ | عند `category=private_events` |

## Frontend — تطبيق العميل

| العنصر | الحالة | الموقع |
|--------|--------|--------|
| أيقونة "المناسبات الخاصة" | ✅ | `categories_screen.dart` |
| فتح Feed بفئة private_events | ✅ | `_openFeed(context, ProviderCategories.privateEvents)` |
| عرض Feed لمقدمي المناسبات | ✅ | `FeedScreen` مع `category=private_events` |
| زر "طلب مناسبة" بجانب الفيديو | ✅ | `feed_video_side_actions.dart` — `_getRequestLabel` يرجع `requestEvent` |
| الانتقال لطلب مناسبة | ⚠️ جزئي | يذهب لـ `RequestChefScreen` — نفس شاشة الطبخ المنزلي (غير مناسبة) |

## Frontend — تطبيق المقدم

| العنصر | الحالة | الموقع |
|--------|--------|--------|
| اختيار "مناسبات خاصة" عند التسجيل | ✅ | `register_screen.dart` |

## Frontend — لوحة الأدمن

| العنصر | الحالة | الموقع |
|--------|--------|--------|
| عرض فئة "المناسبات الخاصة" في قائمة المقدمين | ✅ | `admin_panel/vendors/page.tsx` |

# ٢. ما هو غير منفذ

## Backend

| العنصر | الوصف |
|--------|--------|
| نوع طلب المناسبات في event_requests | جدول `event_requests` يدعم فقط `popular_cooking` و `home_cooking` — لا يوجد `private_events` |
| كيانات عروض المناسبات | لا يوجد جدول/كيانات لـ: باقات بوفيه، حلويات، عصائر، مقدمي خدمة — بأسعار يحددها المقدم |
| أنواع المناسبات | لا يوجد enum أو جدول: زواج، تخرج، ملكة، خطوبة، إلخ |
| أنواع الخدمات للمناسبات | لا يوجد: بوفيه مفتوح، حلويات، عصائر، مقدمي خدمة (ندل، مساعدون) |
| API طلب مناسبة خاصة | لا يوجد endpoint مخصص لطلب مناسبة (نوع المناسبة + الخدمات + عدد الضيوف) |
| API عروض المقدم للمناسبات | لا يوجد endpoint لعروض المقدم (باقات، أسعار، سعة) |

## Frontend — تطبيق العميل

| العنصر | الوصف |
|--------|--------|
| شاشة طلب مناسبة مخصصة | لا توجد شاشة مخصصة — يستخدم RequestChefScreen (الطبخ المنزلي) وهو غير مناسب |
| اختيار نوع المناسبة | لا يوجد: زواج، تخرج، ملكة، خطوبة |
| اختيار الخدمات المطلوبة | لا يوجد: بوفيه، حلويات، عصائر، مقدمي خدمة |
| عدد الضيوف لكل خدمة | لا يوجد |
| فلاتر خاصة بالمناسبات | لا يوجد: نوع المناسبة، عدد الضيوف، التاريخ، نطاق السعر |
| عرض باقات وعروض المقدم | لا يوجد — Feed يعرض menu items فقط (وجبات) وليس باقات مناسبات |

## Frontend — تطبيق المقدم

| العنصر | الوصف |
|--------|--------|
| إدارة عروض المناسبات | لا يوجد: إضافة باقات بوفيه، حلويات، عصائر، مقدمي خدمة — مع الأسعار |
| تحديد أنواع المناسبات التي يخدمها | لا يوجد |
| تحديد السعة (عدد الأشخاص) | لا يوجد |
| عرض طلبات المناسبات | لا يوجد — لا يرى المقدم طلبات "طلب مناسبة" |

# ٣. المتطلبات الكاملة (بدون تكرار)

## Backend

| # | المتطلب | التفاصيل |
|---|---------|----------|
| B1 | إضافة `private_events` لـ event_requests | توسيع enum `request_type` أو إنشاء جدول `private_event_requests` منفصل |
| B2 | جدول عروض المناسبات | `event_offers` أو `vendor_event_packages`: vendor_id, service_type (buffet/desserts/drinks/staff), event_type (wedding/graduation/henna/engagement), price, min_guests, max_guests |
| B3 | جدول طلبات المناسبات | `private_event_requests`: user_id, vendor_id, event_type, event_date, address_id, services (JSON: [{type, guests_count}]), notes, status |
| B4 | API إنشاء طلب مناسبة | `POST /private-event-requests` |
| B5 | API عروض المقدم | `GET /vendors/:id/event-offers` — باقات وأسعار المقدم |
| B6 | API طلبات المقدم | `GET /vendors/private-event-requests` — للمقدم لعرض طلباته وقبولها/رفضها |

## Frontend — تطبيق العميل

| # | المتطلب | التفاصيل |
|---|---------|----------|
| F1 | شاشة طلب مناسبة | شاشة جديدة `RequestPrivateEventScreen` — تظهر عند الضغط على "طلب مناسبة" من Feed المناسبات |
| F2 | اختيار نوع المناسبة | زواج، تخرج، ملكة، خطوبة، أخرى |
| F3 | اختيار الخدمات | بوفيه (عدد أشخاص)، حلويات (عدد أشخاص)، عصائر (عدد أشخاص)، مقدمي خدمة (عدد أشخاص) |
| F4 | التاريخ والوقت والعنوان | مثل الطلبات الأخرى |
| F5 | فلاتر Feed المناسبات | نوع المناسبة، عدد الضيوف، التاريخ، نطاق السعر — في شريط الفلاتر عند category=private_events |
| F6 | عرض عروض المقدم | في صفحة تفاصيل المقدم — باقات بوفيه، حلويات، عصائر، مقدمي خدمة — مع الأسعار |
| F7 | ربط زر "طلب مناسبة" | يفتح `RequestPrivateEventScreen` وليس `RequestChefScreen` |

## Frontend — تطبيق المقدم

| # | المتطلب | التفاصيل |
|---|---------|----------|
| M1 | شاشة إدارة عروض المناسبات | إضافة/تعديل باقات: بوفيه، حلويات، عصائر، مقدمي خدمة — مع السعر والسعة |
| M2 | تحديد أنواع المناسبات | اختيار: زواج، تخرج، ملكة، خطوبة — التي يخدمها |
| M3 | شاشة طلبات المناسبات | عرض الطلبات الواردة، قبول/رفض، تفاصيل كل طلب |

# ٤. ملخص سريع

| الجهة | منفذ | غير منفذ |
|-------|------|----------|
| **Backend** | provider_category، فلتر Feed، تسجيل المقدم | event type، service types، عروض المناسبات، طلبات المناسبات، APIs |
| **Customer App** | أيقونة، Feed، زر "طلب مناسبة" | شاشة طلب مناسبة، فلاتر، عروض المقدم |
| **Vendor App** | اختيار الفئة عند التسجيل | إدارة العروض، طلبات المناسبات |
