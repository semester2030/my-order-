# حالة ميزة الطلب المسبق / الجدولة للطبخ المنزلي

## الميزات التي ناقشناها

1. **وجبات جاهزة الآن** — الطبق جاهز، أضف للسلة، توصيل فوري
2. **طلب مسبق / جدولة** — العميل يطلب طبقاً لوقت محدد (بعد ساعة، غداً الساعة ٢، إلخ)
3. **إعدادات توفر الطباخة** — أيام، أوقات، أو عدد وجبات أسبوعياً
4. **تنسيق الطلبات** — قبول/رفض أو اقتراح وقت بديل

---

## ✅ ما تم تنفيذه (من الكود)

| الميزة | الملف | التفاصيل |
|-------|--------|----------|
| **وجبات جاهزة الآن** | Cart → Order flow | أضف للسلة → الدفع → الطلب يُنشأ فوراً |
| **وقت التوصيل التقديري** | `orders.service.ts` سطر 52–56 | يُحسب تلقائياً: 30–45 دقيقة من الآن (لا يحدده العميل) |
| **isAcceptingOrders** | `vendor.entity.ts` | الطباخة تفتح/تُغلق استقبال الطلبات |
| **isAvailable (للأطباق)** | `menu-item.entity.ts` + `PATCH /vendors/menu/:id/availability` | تفعيل/إيقاف كل طبق |
| **workingHours** | `vendor.entity.ts` سطر 156–162 | حقل JSON (day, open, close) — موجود في الـ entity |
| **workingHours في الإعدادات** | `vendor-web/settings/page.tsx` | حقل نصي عام: "9:00 AM - 11:00 PM" — **لا يُستخدم** في منطق الطلبات أو الجدولة |
| **notes في الطلب** | `CreateOrderDto` | `notes` موجود في الـ DTO — **لكن تطبيق العميل لا يمرره** (`createOrder(defaultAddress.id)` فقط) |

## ❌ ما لم يُنفَّذ (من الكود)

| الميزة | المطلوب | الحالة الحالية |
|--------|---------|----------------|
| **وقت مطلوب من العميل** | العميل يحدد "جاهز بعد ساعة" أو "غداً الساعة ٢" | ❌ لا يوجد `requestedReadyAt` في Order أو CreateOrderDto |
| **اختيار الوقت عند الإضافة للسلة** | اختيار "جاهز الآن" أو "جاهز في وقت محدد" | ❌ لا يوجد — الإضافة للسلة بدون وقت |
| **اختيار الوقت عند الدفع** | اختيار وقت الاستلام قبل إنشاء الطلب | ❌ لا يوجد — الدفع مباشر بدون وقت |
| **أوقات توفر الطباخة** | أيام + أوقات (مثلاً: السبت–الخميس 12–3، 6–9) | ❌ `workingHours` موجود لكن غير مستخدم في منطق الطلبات |
| **عدد الوجبات أسبوعياً** | مثلاً: "١٠ وجبات أسبوعياً" | ❌ لا يوجد `mealsPerWeek` أو ما شابه |
| **حد أدنى لوقت التحضير** | مثلاً: "الاستلام بعد ساعتين كحد أدنى" | ❌ لا يوجد |
| **نوع الطلب** | `ready_now` vs `scheduled` | ❌ لا يوجد `orderType` في Order |
| **قبول/رفض الطلب المسبق** | الطباخة تقبل أو ترفض مع اقتراح وقت | ❌ الطلبات تُنشأ وتُقبل تلقائياً (لا مسار طلب مسبق) |
| **عرض وقت مطلوب للطباخة** | الطباخة ترى "العميل يريده الساعة ٣" | ❌ لا يوجد — الطلبات بدون وقت مطلوب |

## ملخص رقمي

| | العدد |
|---|------|
| **مُنفَّذ بالكامل** | ١ (وجبات جاهزة الآن) |
| **مُنفَّذ جزئياً** (موجود لكن غير مستخدم أو ناقص) | ٣ (workingHours، notes، estimatedDeliveryTime) |
| **غير مُنفَّذ** | ٩ |

## الخلاصة

**الوضع الحالي:** التطبيق يدعم فقط **وجبات جاهزة الآن** — الطلب يُنشأ فوراً ووقت التوصيل يُقدَّر تلقائياً (30–45 دقيقة). لا يوجد أي دعم لطلب مسبق أو جدولة وقت الاستلام.
